### Детальное описание этапов

#### Этап 1: Осознание (Awareness)

| Параметр | Описание |
|----------|----------|
| **Триггер** | Нужно повесить картину / сделать мелкий ремонт |
| **Контекст** | Выходные, дома, не хочет ехать в магазин |
| **Каналы входа** | Пост в домовом чате, рекомендация соседа, ссылка от друга |
| **Барьер** | Не знает о существовании сервиса |
| **Решение** | Виральные посты, интеграция с УК |

**Пользователь думает:**
> "Мне нужен перфоратор на 1 час. Покупать за 8000₽ глупо. Может, в чате спросить?"

#### Этап 2: Поиск (Search)

| Параметр | Описание |
|----------|----------|
| **Действие** | Открывает WebApp, смотрит каталог |
| **Ожидание** | Найти нужное быстро и рядом |
| **Боль** | Мало вариантов, неудобный поиск |
| **Решение** | Умный поиск, категории, фильтры |

**Touchpoints:**
1. Клик по ссылке в чате → WebApp
2. Выбор категории "Инструменты"
3. Поиск "перфоратор"
4. Фильтр по цене

**Метрики этапа:**
- Time to first listing view: <30 сек
- Search success rate: >80%

#### Этап 3: Выбор (Evaluation)

| Параметр | Описание |
|----------|----------|
| **Действие** | Сравнивает варианты, читает описания |
| **Ожидание** | Понять качество и надёжность |
| **Боль** | Недостаточно информации, страх повреждения |
| **Решение** | Фото, описание, рейтинги, страхование |

**Что оценивает:**
- Цена за сутки
- Размер депозита
- Рейтинг вещи и владельца
- Фото (качество, состояние)
- Расстояние (этаж/подъезд)

**Метрики этапа:**
- Listings viewed per session: 3-5
- View-to-book rate: 10-15%

#### Этап 4: Бронирование (Booking)

| Параметр | Описание |
|----------|----------|
| **Действие** | Выбирает даты, оплачивает |
| **Ожидание** | Простой и безопасный процесс |
| **Боль** | Сложная оплата, непонятно что дальше |
| **Решение** | 3 клика до оплаты, YooKassa, эскроу |

**Шаги:**
1. Клик "Забронировать"
2. Выбор дат в календаре
3. Просмотр расчёта (аренда + депозит)
4. Переход к оплате (YooKassa)
5. Возврат с подтверждением

**Метрики этапа:**
- Checkout started to completed: 70%
- Payment success rate: 95%
- Time to complete booking: <2 мин

#### Этап 5: Аренда (Usage)

| Параметр | Описание |
|----------|----------|
| **Действие** | Получает вещь, использует, возвращает |
| **Ожидание** | Вещь работает, возврат простой |
| **Боль** | Сложно найти владельца, вещь не работает |
| **Решение** | Чат в Telegram, инструкции, напоминания |

**Сценарий передачи:**
1. Бронирование подтверждено → контакт владельца
2. Пишет владельцу в Telegram
3. Договариваются о месте/времени
4. Встречаются, передают вещь
5. Использует
6. Связывается для возврата
7. Возвращает

**Метрики этапа:**
- Time from confirmation to pickup: <4 часа
- Issues during rental: <5%
- On-time return rate: >95%

#### Этап 6: После аренды (Post-rental)

| Параметр | Описание |
|----------|----------|
| **Действие** | Возвращает, получает депозит, оставляет отзыв |
| **Ожидание** | Быстрый возврат денег, простой отзыв |
| **Боль** | Долгий возврат, забывает оставить отзыв |
| **Решение** | Мгновенный возврат депозита, напоминание |

**Метрики этапа:**
- Review rate: >50%
- Average rating: 4.5+
- Repeat booking rate (30 days): 15%

### Детальное описание этапов

#### Этап 1: Осознание (Awareness)

| Параметр | Описание |
|----------|----------|
| **Триггер** | Вещи простаивают, хочет заработать |
| **Контекст** | Разбор кладовки, видит посты в чате |
| **Каналы входа** | Домовой чат, сарафанное радио |
| **Барьер** | Страх сложности, порчи вещей |
| **Решение** | Простой UX, гарантии безопасности |

**Пользователь думает:**
> "Велосипед стоит 3 года. Продавать жалко, но хоть заработать..."

#### Этап 2: Создание листинга (Listing)

| Параметр | Описание |
|----------|----------|
| **Действие** | Фотографирует, заполняет форму |
| **Ожидание** | Быстро и просто |
| **Боль** | Не знает какую цену ставить |
| **Решение** | Подсказка цены по категории, простая форма |

**Шаги создания:**
1. Клик "Добавить вещь"
2. Загрузка фото (1-5)
3. Название (автокомплит)
4. Описание (шаблон)
5. Категория (выбор)
6. Цена (подсказка рынка)
7. Депозит (default = 2x цена)
8. Публикация

**Метрики этапа:**
- Listing creation time: <5 мин
- Completion rate: >70%
- Photos per listing: 3+

#### Этап 3: Ожидание заявок (Waiting)

| Параметр | Описание |
|----------|----------|
| **Действие** | Ждёт, проверяет статистику |
| **Ожидание** | Быстрые заявки |
| **Боль** | Нет заявок, не понимает почему |
| **Решение** | Статистика просмотров, советы по улучшению |

**Уведомления:**
- "Ваш листинг просмотрели 5 раз"
- "Совет: добавьте больше фото"
- "В вашей категории средняя цена 400₽/день"

**Метрики этапа:**
- Time to first booking: <7 дней
- Views per listing/day: 2-5

#### Этап 4: Получение заявки (Request)

| Параметр | Описание |
|----------|----------|
| **Действие** | Получает push, решает принять/отклонить |
| **Ожидание** | Понять, можно ли доверять арендатору |
| **Боль** | Ненадёжный арендатор |
| **Решение** | Рейтинг и история арендатора |

**Информация в заявке:**
- Имя и фото арендатора
- Рейтинг (звёзды + количество отзывов)
- Даты аренды
- Сумма (что получит владелец после комиссии)
- Комментарий арендатора (если есть)

**Метрики этапа:**
- Response time: <4 часа
- Approval rate: >80%

#### Этап 5: Передача и возврат (Rental)

| Параметр | Описание |
|----------|----------|
| **Действие** | Передаёт вещь, принимает назад |
| **Ожидание** | Всё пройдёт гладко |
| **Боль** | Повреждение вещи |
| **Решение** | Депозит, страхование, чёткий процесс |

**Процесс передачи:**
1. Подтверждение заявки → контакт арендатора
2. Договорённость о встрече
3. Передача вещи
4. (Аренда)
5. Возврат вещи
6. Проверка состояния
7. Клик "Вещь возвращена" → выбор состояния

**Метрики этапа:**
- Issue rate: <5%
- Damage claims: <2%

#### Этап 6: После аренды (Post-rental)

| Параметр | Описание |
|----------|----------|
| **Действие** | Получает деньги, оставляет отзыв |
| **Ожидание** | Быстрый перевод |
| **Боль** | Долгий вывод средств |
| **Решение** | T+1 выплата, прозрачный баланс |

**Метрики этапа:**
- Payout time: T+1
- Review rate: >60%
- Repeat listing activity: >70%

## 3. Момент истины (Moments of Truth)

## 5. Pain Points & Opportunities

### Карта болей

```
                        SEVERITY
               Low ◄─────────────────► High
           │
     High  │  • Мало вещей        • Сломал вещь
           │    в каталоге         (владелец)
           │                      • Не вернули
           │  • Неудобный           вещь
FREQUENCY  │    поиск
           │                      • Владелец
           │  • Долгий ответ        не отвечает
           │    владельца
           │
     Low   │  • Сложная           • Спор по
           │    регистрация         депозиту
           │
```

### Возможности улучшения

| Pain Point | Opportunity | Impact | Effort |
|------------|-------------|--------|--------|
| Мало вещей | Seeding, инвайты | High | Medium |
| Долгий ответ | Автоподтверждение (trusted) | Medium | Low |
| Сложный поиск | AI-рекомендации | Medium | High |
| Страх поломки | Расширенное страхование | High | Medium |
| Споры по депозиту | Автоматизация решений | Medium | High |
