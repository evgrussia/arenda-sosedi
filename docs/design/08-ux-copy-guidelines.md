# UX Copy Guidelines — Арендо-Соседи

## Введение

Документ определяет принципы и стандарты написания UX-текстов для платформы Арендо-Соседи. Цель — создать консистентный, понятный и дружелюбный голос продукта.

---

## 1. Voice & Tone Foundation

### 1.1 Голос бренда (Voice)

```yaml
Характеристики:
  - Дружелюбный: как хороший сосед
  - Понятный: без жаргона и сложных терминов
  - Помогающий: ориентирован на решение задачи
  - Уверенный: знает как помочь
  - Человечный: не роботизированный

НЕ:
  - Формальный или бюрократический
  - Снисходительный
  - Чрезмерно casual/молодёжный
  - Агрессивно-продающий
```

### 1.2 Тон (Tone)

Тон меняется в зависимости от контекста:

| Контекст | Тон | Пример |
|----------|-----|--------|
| Успех | Радостный, поздравляющий | "Отлично! Заявка отправлена" |
| Ошибка | Спокойный, помогающий | "Не удалось загрузить. Попробуйте ещё раз" |
| Предупреждение | Серьёзный, но не пугающий | "Проверьте данные перед отправкой" |
| Ожидание | Спокойный, информативный | "Ожидаем подтверждения владельца" |
| Onboarding | Приветливый, мотивирующий | "Добро пожаловать в соседское сообщество!" |
| Финансы | Чёткий, надёжный | "Залог 1 000 ₽ вернётся после возврата" |

## 2. Writing Principles

### 2.1 Ясность

```markdown
✓ Пишите просто и понятно
✓ Одна мысль — одно предложение
✓ Избегайте двусмысленности
✓ Используйте активный залог

Плохо:  "Операция по бронированию была успешно завершена системой"
Хорошо: "Бронирование подтверждено"
```

### 2.2 Краткость

```markdown
✓ Убирайте лишние слова
✓ Каждое слово должно работать
✓ Не повторяйтесь

Плохо:  "Пожалуйста, нажмите на кнопку 'Далее' для того, чтобы продолжить"
Хорошо: "Нажмите 'Далее'"
```

### 2.3 Полезность

```markdown
✓ Говорите что делать, а не что случилось
✓ Объясняйте почему (если нужно)
✓ Давайте выход из ситуации

Плохо:  "Ошибка 500"
Хорошо: "Не удалось загрузить. Проверьте интернет и попробуйте снова"
```

### 2.4 Позитивность

```markdown
✓ Фокус на возможностях, не на ограничениях
✓ Избегайте негативных формулировок

Плохо:  "Вы не можете арендовать без регистрации"
Хорошо: "Зарегистрируйтесь, чтобы начать арендовать"
```

## 3. Grammar & Style

### 3.1 Обращение к пользователю

```yaml
Правило: На "вы" (с маленькой буквы)

Примеры:
  ✓ "Ваше объявление опубликовано"
  ✓ "Войдите в аккаунт"
  ✗ "Ваше объявление опубликовано" (заглавная В — устаревший стиль)
```

### 3.2 Числа и валюта

```yaml
Числа:
  - От 1 до 9: словами (один, два, три)
  - От 10: цифрами (10, 15, 100)
  - Исключение: в UI всегда цифрами

Валюта:
  - Формат: 1 000 ₽ (пробел-разделитель, символ после)
  - Без копеек в UI
  - "рублей" в длинных текстах

Даты:
  - Короткий: 5 фев
  - Полный: 5 февраля 2025
  - Период: 5 — 7 февраля
```

### 3.3 Время

```yaml
Формат:
  - 14:30 (24-часовой формат)
  - "сегодня", "завтра", "вчера" где возможно
  - "2 часа назад", "через 3 дня"

Примеры:
  - "Ответьте в течение 24 часов"
  - "Создано вчера в 14:30"
```

### 3.4 Заглавные буквы

```yaml
Правила:
  - Заголовки: С заглавной первое слово
  - Кнопки: С заглавной первое слово
  - Labels: С заглавной первое слово
  - Предложения: Стандартная пунктуация

Примеры:
  Заголовок: "Мои объявления"
  Кнопка: "Опубликовать"
  Label: "Цена за день"

НЕ делать:
  - ВСЕ ЗАГЛАВНЫЕ (кроме аббревиатур)
  - Каждое Слово С Заглавной
```

### 3.5 Пунктуация

```yaml
Точки:
  - Заголовки: без точки
  - Кнопки: без точки
  - Списки: без точки (если пункты короткие)
  - Полные предложения: с точкой

Восклицательные знаки:
  - Используйте редко (success states)
  - Не более одного подряд

Многоточие:
  - Для загрузки: "Загрузка..."
  - Для усечённого текста: "Описание товара..."
```

## 4. UI Text Patterns

### 4.1 Buttons

```yaml
Правила:
  - Глагол в инфинитиве или повелительном наклонении
  - Конкретное действие
  - Короткий текст (1-3 слова)

Primary Actions:
  ✓ "Арендовать"
  ✓ "Опубликовать"
  ✓ "Оплатить"
  ✓ "Подтвердить"

Secondary Actions:
  ✓ "Отмена"
  ✓ "Назад"
  ✓ "Пропустить"
  ✓ "Позже"

Destructive:
  ✓ "Удалить"
  ✓ "Отклонить"

С контекстом:
  ✓ "Оплатить 1 675 ₽"
  ✓ "Показать 14 результатов"
```

### 4.2 Labels & Placeholders

```yaml
Labels:
  - Существительное или короткая фраза
  - Без двоеточия
  - Пример: "Название", "Цена за день"

Placeholders:
  - Пример формата или подсказка
  - Пример: "Например: Дрель Bosch", "От 100 ₽"

Helper Text:
  - Дополнительная информация под полем
  - Пример: "Минимум 20 символов"
```

### 4.3 Empty States

```yaml
Структура:
  - Заголовок (что произошло)
  - Описание (почему и что делать)
  - CTA (действие)

Примеры:

Нет объявлений в ЖК:
  Title: "Пока пусто"
  Desc: "В вашем ЖК ещё никто не разместил объявлений"
  CTA: "Станьте первым!"

Нет результатов поиска:
  Title: "Ничего не нашлось"
  Desc: "Попробуйте изменить запрос или фильтры"
  CTA: "Сбросить фильтры"

Нет чатов:
  Title: "Здесь будут ваши чаты"
  Desc: "Начните переписку с соседями, арендуя или сдавая вещи"
  CTA: "Перейти в каталог"
```

### 4.4 Notifications & Alerts

```yaml
Push Notifications:
  Формат: "[Событие]: [Контекст]"
  Примеры:
    - "Новая заявка на Дрель Bosch"
    - "Бронирование подтверждено"
    - "Напоминание: вернуть Велосипед завтра"

In-App Alerts:
  Success:
    - "Готово! [Что сделано]"
    - "Объявление опубликовано"

  Error:
    - "[Что не удалось]. [Что делать]"
    - "Не удалось отправить. Проверьте интернет"

  Info:
    - Нейтральная информация
    - "Владелец ответит в течение 24 часов"
```

### 4.5 Confirmations

```yaml
Структура:
  Title: Вопрос или утверждение
  Desc: Последствия действия (если важно)
  Actions: [Отмена] [Подтвердить]

Примеры:

Удаление:
  Title: "Удалить объявление?"
  Desc: "Это действие нельзя отменить"
  Actions: [Отмена] [Удалить]

Отмена бронирования:
  Title: "Отменить бронирование?"
  Desc: "Вам вернётся 75% суммы согласно политике отмены"
  Actions: [Назад] [Отменить бронирование]
```

## 5. Specific Contexts

### 5.1 Onboarding

```yaml
Welcome:
  Title: "Добро пожаловать в Арендо-Соседи!"
  Desc: "Арендуйте вещи у соседей — быстро, рядом, безопасно"

Tutorial Step 1:
  Title: "Находите нужное рядом"
  Desc: "Смотрите что сдают соседи в вашем доме"

Tutorial Step 2:
  Title: "Арендуйте в пару кликов"
  Desc: "Бронируйте, оплачивайте и договаривайтесь в чате"

Tutorial Step 3:
  Title: "Безопасно и с гарантией"
  Desc: "Защита платежей и поддержка 24/7"
```

### 5.2 Финансы

```yaml
Tone: Чёткий, прозрачный, надёжный

Примеры:

Баланс:
  "Баланс: 2 450 ₽"
  "Доступно к выводу: 1 200 ₽"
  "В ожидании: 1 250 ₽" (с tooltip: "Средства за активные аренды")

Комиссия:
  "Сервисный сбор: 75 ₽"
  (не "комиссия платформы")

Залог:
  "Залог: 1 000 ₽"
  "Залог вернётся в течение 24 часов после возврата вещи"

Вывод:
  "Вывести на карту"
  "Средства поступят в течение 1-3 рабочих дней"
```

### 5.3 Безопасность и доверие

```yaml
Tone: Спокойный, уверенный

Примеры:

Верификация:
  "Ваш адрес подтверждён"
  "Верифицированный сосед"

Платежи:
  "Безопасная оплата"
  "Деньги хранятся на защищённом счёте до завершения аренды"

Поддержка:
  "Поддержка 24/7"
  "Мы поможем решить любой вопрос"
```

### 5.4 Споры

```yaml
Tone: Нейтральный, справедливый

Примеры:

Открытие спора:
  Title: "Что-то пошло не так?"
  Desc: "Опишите проблему, и мы поможем её решить"

Ожидание ответа:
  "Ожидаем ответ второй стороны"
  "У владельца есть 48 часов на ответ"

Решение:
  "Спор решён в вашу пользу"
  "Залог возвращён полностью"
```

## 6. Content Matrix

### 6.1 Действия пользователя

| Действие | Success | Error | Loading |
|----------|---------|-------|---------|
| Регистрация | "Добро пожаловать!" | "Не удалось создать аккаунт" | "Создаём аккаунт..." |
| Публикация | "Объявление опубликовано" | "Не удалось опубликовать" | "Публикуем..." |
| Бронирование | "Заявка отправлена" | "Не удалось забронировать" | "Отправляем заявку..." |
| Оплата | "Оплата прошла успешно" | "Оплата не прошла" | "Обрабатываем платёж..." |
| Отправка сообщения | — | "Не удалось отправить" | — |
| Подтверждение возврата | "Аренда завершена" | "Не удалось подтвердить" | "Подтверждаем..." |

### 6.2 Статусы

| Статус | Для арендатора | Для владельца |
|--------|----------------|---------------|
| Pending | "Ожидает подтверждения" | "Новая заявка" |
| Confirmed | "Подтверждено" | "Вы подтвердили" |
| Active | "В аренде" | "Сдано" |
| Returning | "Возврат" | "Ожидает возврата" |
| Completed | "Завершено" | "Завершено" |
| Cancelled | "Отменено" | "Отменено" |
| Disputed | "На рассмотрении" | "На рассмотрении" |

## 7. Glossary (UX Copy)

### 7.1 Предпочтительные термины

| Используем | Не используем |
|------------|---------------|
| Объявление | Листинг, оффер |
| Арендовать | Снять, взять напрокат |
| Сдать в аренду | Разместить, выложить |
| Владелец | Арендодатель, хозяин |
| Арендатор | Съёмщик, клиент |
| Залог | Депозит |
| Сервисный сбор | Комиссия платформы |
| Заявка | Запрос на бронирование |
| Подтвердить | Одобрить, акцептовать |

### 7.2 Избегаемые слова

```yaml
Избегать:
  - Техжаргон: "транзакция", "эскроу", "пейаут"
  - Юридизмы: "акцептовать", "оферта"
  - Негатив: "проблема", "ошибка" (где можно заменить)
  - Канцелярит: "осуществить", "произвести"
  - Anglicisms без нужды: "апрувить", "чекнуть"
```

## 8. Content QA Checklist

```markdown
## Перед публикацией проверить:

### Ясность
- [ ] Понятно с первого прочтения?
- [ ] Нет двусмысленности?
- [ ] Одна мысль — одно предложение?

### Стиль
- [ ] Соответствует tone of voice?
- [ ] Активный залог?
- [ ] На "вы" с маленькой буквы?

### Формат
- [ ] Правильные числа и валюта?
- [ ] Пунктуация по гайдам?
- [ ] Заглавные буквы корректны?

### UX
- [ ] Говорит что делать (а не что случилось)?
- [ ] Есть выход из ситуации?
- [ ] Позитивная формулировка?

### A11y
- [ ] Понятно без контекста (для SR)?
- [ ] Нет только emoji/иконок?
```
